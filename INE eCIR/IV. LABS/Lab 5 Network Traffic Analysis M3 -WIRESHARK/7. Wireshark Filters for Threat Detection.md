1. HTTP GET Flooding
Threat: Massive HTTP GET requests overload server capacity.
Filter:

wireshark
Copy code
http.request.method == "GET"
Check: Abnormal number of GET requests from same IP or strange User-Agent patterns.

2. DNS Tunneling
Threat: DNS used for covert data exfiltration.
Filter:

wireshark
Copy code
dns
Check: Abnormal DNS payload sizes, repeated requests, long response times.

3. SMB Brute Force
Threat: Multiple SMB login attempts to guess credentials.
Filter:

wireshark
Copy code
smb.cmd == 0x73
Check: Failed login status codes, repeated authentication attempts.

4. SYN Flood Attack
Threat: Flooding server with SYN packets to exhaust resources.
Filter:

wireshark
Copy code
tcp.flags.syn == 1 && tcp.flags.ack == 0
Check: High SYN count with missing ACKs or delayed ACKs.

5. DNS Amplification
Threat: Small requests trigger large DNS responses to overwhelm victim.
Filter:

wireshark
Copy code
dns.qry.name
Check: Large DNS response sizes or suspicious source addresses.

6. ICMP Flooding (Ping of Death)
Threat: Massive ICMP Echo Requests to consume bandwidth.
Filter:

wireshark
Copy code
icmp
Check: High frequency ICMP packets or oversized payloads (>64 bytes).

Filter:

wireshark

wireshark
